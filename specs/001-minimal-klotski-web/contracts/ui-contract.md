# UI 合同: 极简华容道纯网页版

## 页面入口

- 单页入口: `/`
- 不要求登录、路由切换或后端接口。

## 必需界面区域

| 区域 | 目的 | 最低要求 |
|------|------|----------|
| 顶层容器 | 承载应用布局 | 使用 `#app` 或等效根节点 |
| 棋盘区域 | 展示 4x5 华容道棋盘 | 具备稳定宽高比和居中布局 |
| HUD 区域 | 展示关卡、步数、时间 | 不得遮挡棋盘主要交互区域 |
| 控制区 | 提供关卡切换、撤销、重开、反馈开关 | 触屏可点击, 文字或图标可理解 |
| 反馈层 | 展示加载、错误、通关提示 | 不阻断已完成对局的阅读 |

## 建议测试钩子

| 元素 | 建议选择器 |
|------|------------|
| 棋盘容器 | `[data-testid="board"]` |

| 关卡选择 | `[data-testid="level-select"]` |

| 撤销按钮 | `[data-testid="undo-button"]` |
| 重开按钮 | `[data-testid="restart-button"]` |

| 错误态 | `[data-testid="error-state"]` |

## 交互合同

- 玩家通过按压棋子并沿单一主方向滑动后释放来尝试移动。
- 系统只接受当前激活的一个手势会话; 并发或多点输入不得改变棋盘一致性。
- 合法手势必须触发一次自动吸附移动和一步计数更新。
- 非法手势、阈值不足或被阻挡的手势必须清除按压态并让棋子回到原始位置。
- `undo` 仅在存在历史记录时可执行; 不可执行时按钮需提供禁用态或等效反馈。
- `restart` 必须恢复当前关卡初始布局、步数和计时。
- 关卡切换必须销毁旧会话并创建新会话, 不得残留旧棋盘位置。

## 状态合同

| 状态 | 触发时机 | UI 要求 |
|------|----------|---------|
| `loading` | 初始加载或切换关卡 | 提供轻量加载反馈, 棋盘区域不闪烁错位 |
| `empty` | 关卡库为空或默认关卡不可用 | 展示空状态说明、重新加载入口和非阻断返回路径 |
| `ready` | 关卡已加载、尚未移动 | 展示棋盘和初始 HUD |
| `active` | 存在进行中的对局 | HUD 实时更新步数和用时 |
| `won` | 目标棋子抵达出口 | 展示通关层、最终步数和用时 |
| `error` | 关卡或资源加载失败 | 展示可恢复错误提示和重新加载入口 |

## 响应式与可访问性合同

- 手机、平板、桌面均不得出现横向滚动来完成主流程。
- 棋盘必须在任何断点下保持完整可见与视觉居中。
- 所有按钮必须具备可见焦点态和可读标签。
- `level-select`, `undo-button`, `restart-button`, `sound-toggle`, `haptics-toggle`
  都必须具备可见焦点态、可读标签和可通过键盘理解的状态变化。
- 状态文案和通关反馈必须可被屏幕阅读器感知, 建议通过 `aria-live` 或等效方式实现。
- 音效和震动必须默认克制且可关闭, 关闭后 UI 需同步反映当前设置。
