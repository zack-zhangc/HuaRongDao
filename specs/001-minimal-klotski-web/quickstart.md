# Quickstart: 极简华容道纯网页版

## 前置条件

- Node.js 22 LTS
- npm
- 支持现代 ES Modules 的浏览器

## 本地启动

1. 在仓库根目录安装开发依赖:

   ```bash
   npm install
   ```

2. 启动静态开发服务:

   ```bash
   npm run dev
   ```

3. 在浏览器中打开本地地址, 确认默认关卡能正常渲染。

## 质量门控

按以下顺序运行:

```bash
npm run format:check
npm run lint
npm run typecheck
npm run test:unit
npm run test:e2e
npm run build
```

## 手动验收路径

### 主流程

1. 打开页面并确认 2 秒内可看到并操作默认棋盘。
2. 对任一可移动棋子执行一次按压后滑动并释放, 确认合法移动会吸附到位并增加步数。
3. 对被阻挡棋子执行非法方向滑动, 确认棋子回弹且步数不增加。
4. 完成一局胜利, 确认展示通关层、最终步数和用时。

### 进程与容错

1. 切换到另一经典关卡, 确认布局、难度和计时重置。
2. 完成若干步移动后执行撤销, 确认棋盘和步数回退。
3. 点击重开, 确认恢复初始布局。
4. 在无可撤销记录时点击撤销, 确认出现非阻断反馈。
5. 加载空关卡测试夹具或让关卡库返回空集合时, 确认页面展示空状态说明和重新加载入口。

### 反馈与适配

1. 在支持音效和震动的设备上完成合法移动, 确认反馈生效。
2. 关闭音效和震动开关后再次移动, 确认不再触发对应反馈。
3. 在手机、平板和桌面宽度下检查棋盘始终居中且无横向滚动。
4. 旋转设备或调整窗口大小, 确认 1 秒内恢复为可继续操作的布局。
5. 在 Chromium、WebKit 和 Safari 手动验收中分别执行主流程, 确认视口与手势表现一致。

- 使用浏览器 Performance API 标记首屏可玩时间、手势反馈延迟和重布局恢复时间。
- 通过 Playwright 场景脚本重复执行主流程, 记录:
  - 首屏可玩时间 <= 2s
  - 合法手势 50ms 内出现反馈
  - 吸附/回弹 250ms 内完成
  - 重布局 1s 内恢复
